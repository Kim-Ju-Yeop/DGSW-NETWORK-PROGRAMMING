


1. TCP와 UDP의 차이점을 설명하시오

TCP는 연속성보다 신뢰성있는 전송이 중요할 때에 사용하는 프로토콜이며,
UDP는 TCP보다 속도가 빠르며 네트워크 부하가 적다는 장점이 있지만, 신뢰성있는 데이터 전송을 보장하지는 않습니다.
그렇기 때문에 신뢰성보다는 연속성이 중요한 서비스의 예를 들면 실시간 서비스에 자주 사용됩니다.



2. Ip와 host를 설명하시오

- Ip의 개념

컴퓨터가 인터넷을 통해서 통신하기 위해서는 반드시 IP 주소를 가져야한다.
두 대의 이상의 컴퓨터가 서로 통신할 때 클라이언트는 웹브라우저에게 서버의 Ip 주소를 입력하여 접속할 수 있다.
서버는 응답하기 위해서 요청한 컴퓨터의 Ip를 알아야 한다. 클라이언트가 정보를 요청할 때 자신의 주소를 담아서 보낸다.
즉 Ip 주소는 컴퓨터 네트워크에서 장치들이 서로를 인식하고 통신을 하기 위해서 사용하는 특수한 번호를 의미한다.

- host의 개념

네트워크에 연결된 각각의 장치들을 host라고 부른다.



3. URI와 URL에 대해서 설명하시오.

- URI의 개념

인터넷에 있는 자원을 나타내는 유일한 주소이다.
URI의 존재는 인터넷에서 요구되는 기본조건으로서 인터넷 프로토콜에 항상 붙어 다닌다.

- URL의 개념

네트워크 상에서 자원이 어디 있는지를 알려주기 위한 규약이다.



4. 프로토콜이란 무엇인가?

통신 프로토콜 또는 통신 규약은 컴퓨터나 원거리 장비 사이에서 메세지를 주고 받는 양식과 규칙의 체계이다.
통신 프로토콜은 신호 체계, 인증, 그리고 오류 감지 및 수정 기능을 포함할 수 있다.
즉 프로토콜은 데이터를 주고 받는 상호간에 미리 약속된 규칙, 규약이라고 할 수 있습니다.



5. PORT란 무엇인가?

포트 번호는 TCP와 UDP가 상위 계층에 제공하는 주소 표현 방식이다. 
클라이언트와 서버 간의 통신이 이루어질 때 클라이언트는 서버의 IP주소와 포트 번호를 필요로한다.



6. 쓰레드 동작하는 방법에 대해서 설명하시오.

Java언어에서 Thread 클래스와 Runnable 인터페이스를 통해 쓰레드를 구현할 수 있습니다.
쓰레드를 구현하려는 클래스에서 Thread 클래스는 extends Runnable 인터페이스는 implements 예약어를 통해 구현 가능합니다.
각 클래스에 오버라이딩된 run 메서드를 구현한다면 사용자가 원하는 방식대로 동기, 비동기 형식으로 이용할 수 있습니다.



7. ServerSocket과 Socket을 설명하시오.

- ServerSocket의 개념

서버 프로그램을 개발할 때 쓰이는 클래스입니다,
이를 활용해 클라이언트들이 서버에 접속할 수 있도록 도와준다.

- Socket의 개념

TCP 소캣은 java.net.Socket 클래스를 의미한다.
소캣 클래스는 서버에 접속하기 위한 클라이언트를 구현하는데 주로 사용된다.



8. DataGramSocket과 DataGramPacket에 대해 설명하시오.

- DataGramSocket의 개념

데이터 그램 소켓은 패킷 전달 서비스의 송신점 또는 수신점입니다. 데이터 그램 소켓상에서 송신 또는 수신하는 각 패킷은 각각 다른 주소로 경로가 지정됩니다. 어느 머신으로부터 다른 머신에 복수의 패킷을 송신하는 경우, 각 패킷은 다른 경로에서 송신될 가능성이 있어, 행선지에는 임의의 순서로 도달할 가능성이 있습니다.

- DataGramPacket

데이터 그램 패킷은 무접속 패킷 배포 서비스를 구현할 때에 사용합니다. 각 메세지는 패킷내에 포함되어 있는 정보만을 기본으로 어느 머신으로부터 다른 머신에 송신됩니다. 어느 머신으로부터 다른 머신에 복수의 패킷을 송신하는 경우, 각 패킷은 다른 경로에서 송신될 가능성이 있어, 행선지에는 무작위의 순서로 도달할 가능성이 있습니다. 패킷 전달은 보증되지 않습니다.
